Buttons

  • It is important to know when to use a link and when to use a button. A button does an action while a link takes you to another URL.
  • A <button> element does not need anything special to work. ARIA's role="button" is already assigned implicitly. So if you need a button, use a true <button> element.
  • Use <button> when you can, but it is possible to use other elements as long as you add role="button" and add JavaScript to replicate the button functionality.
  • Remember that <button> defaults to type="submit", so adding type="button" helps you avoid accidentally submitting forms.
  • Your button needs a descriptive name. Add class="visuallyhidden" with descriptive text to give more context to the button's purpose.
  • If the button only contains an <img> element, make sure to set its alt attribute. If it only contains an icon use aria-label.

WCAG 2.0 Guidelines

1.1.1 Non-text Content - All non-text content that is presented to the user has a text alternative that serves the equivalent purpose (some exceptions). (Level A)

Note: If non-text content is pure decoration, is used only for visual formatting, or is not presented to users, then it does not need text alternatives.
